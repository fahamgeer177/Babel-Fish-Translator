# Babel Fish Translator 🌍🎙️

A Flask-based voice assistant that listens in English, translates to Spanish when selected, and speaks the response back.

This version runs with local/open-source AI components:
- 🧠 Speech-to-Text: OpenAI Whisper
- 🌐 Translation: Helsinki-NLP (Hugging Face Transformers)
- 🔊 Text-to-Speech: gTTS

## ✨ Features

- 🎤 Record audio directly from the browser
- ⌨️ Send typed messages in the same chat window
- 📝 Automatic speech transcription with Whisper
- 🌍 English to Spanish translation when a Spanish voice is selected
- 🔉 Audio playback for both user recordings and assistant replies
- 🌓 Light/Dark mode toggle in the UI
- 🐳 Docker support for containerized runs

## 🏗️ Project Structure

- `server.py`: Flask routes and API endpoints
- `worker.py`: Speech recognition, translation, and text-to-speech logic
- `templates/index.html`: Main web page
- `static/script.js`: Frontend recording/chat behavior
- `static/style.css`: UI styling
- `setup_local.py`: Optional setup helper script
- `Dockerfile`: Container build and runtime definition

## ✅ Prerequisites

- Python 3.11 recommended
- pip (latest)
- ffmpeg installed on your machine (required by pydub/audio conversion)
- Microphone permission enabled in your browser

## 🚀 Quick Start (Local)

1. Clone the repository and enter the project folder.
2. Create and activate a virtual environment.
3. Install dependencies:

```bash
pip install -r requirements.txt
```

4. Start the app:

```bash
python server.py
```

5. Open:

```text
http://localhost:8000
```

## ⚙️ Optional First-Time Setup Script

If you want a guided setup that also pre-downloads models:

```bash
python setup_local.py
```

## 🐳 Run with Docker

```bash
docker build -t babel-fish-translator .
docker run -p 8000:8000 babel-fish-translator
```

Then open http://localhost:8000.

## 🔌 API Endpoints

- `GET /`
  - Serves the web interface.

- `POST /speech-to-text`
  - Request body: raw audio bytes
  - Response: JSON with transcribed text

- `POST /process-message`
  - Request body JSON:
    - `userMessage` (string)
    - `voice` (string)
  - Response JSON:
    - `watsonxResponseText` (string)
    - `watsonxResponseSpeech` (base64-encoded audio)

## 🧭 Behavior Notes

- If a Spanish voice is selected in the dropdown, the message is translated to Spanish.
- If English/default voice is selected, the text is returned in English.
- Voice IDs in the UI are legacy-style labels; audio output is generated by gTTS language selection.

## 🛠️ Troubleshooting

- No microphone capture:
  - Check browser permissions for microphone access.

- Audio processing errors:
  - Confirm ffmpeg is installed and available in PATH.

- Slow first request:
  - Normal on first run while Whisper/translation models are downloaded and cached.

- Memory pressure on low-RAM systems:
  - Change Whisper model from `base` to `tiny` in `worker.py`.
